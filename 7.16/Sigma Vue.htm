Current Version

Vue 2.6.0 and PrimeVue 1.0.0

Getting Started

Sigma is an application template for Vue based on the Vue CLI that provides out-of-the-box standard tooling for Vue projects. To get started, clone the repository from GitHub and install the dependencies with npm or yarn.

npm install
or 
yarn

Next step is running the application using the serve script and navigate to http://localhost:8080/ to view the application. That is it, you may now start with the development of your application using the Sigma template.

npm run serve

Vue CLI Scripts

Following commands are derived from create-app-app.

"npm run serve": Starts the development server
"npm run build": Builds the application for deployment.
"npm run lint": Executes the lint checks.
"npm run test:unit": Runs the tests.

Structure

Sigma consists of 2 main parts; the application layout and the resources. App.vue inside src folder is the main component containing the template for the base layout whereas required resources such as SASS structure for the layout are placed inside the src/assets/layout folder.

Templates

Main layout is the template of the App.vue, it is divided into a couple of child components such as topbar, profile, menu and footer. Here is template of the App.vue component that implements the logic such as menu state, layout modes and so on. 


<template>
    <div :class="containerClass" @click="onWrapperClick">
        <AppTopBar @menu-toggle="onMenuToggle" />

        <div :class="sidebarClass" @click="onSidebarClick">
            <div class="layout-logo">
                <img alt="Logo" :src="logo" />
            </div>

            <AppProfile />
            <AppMenu :model="menu" @menuitem-click="onMenuItemClick" />
        </div>

        <div class="layout-main">
            <router-view />
        </div>

        <AppFooter />
    </div>
</template>

Dependencies

Dependencies of Sigma are listed below and needs to be added to package.json. Sigma has no direct dependency, even PrimeVue components are an optional dependency..


{
    "primevue": "^1.0.0-rc.6",         //optional: PrimeVue components
    "primeicons": "2.0.0",             //optional: Icons
    "primeflex": "1.0.0",              //optional: Grid system
}

Nova Theme

Sigma uses the free Nova-Light which is a free theme distributed within PrimeVue, however it can be used with any PrimeVue theme as well.

Menu Modes

Menu has 2 modes, static and overlay. Main layout container element in App.vue is used to define which mode to use by adding specific classes. List below indicates the style classes for each mode.

  • Static: "layout-wrapper layout-static"
  • Overlay: "layout-wrapper layout-overlay"

For example to create an overlay menu, the div element should be in following form;


<div class="layout-wrapper layout-static">

It is also possible to leave the choice to the user by keeping the preference at a component and using an expression to bind it so that user can switch between modes. Sample application has an example implementation of such use case with a computed property. Refer to App.vue for an example.

Menu Color Scheme

There are two alternatives as the menu colors schemes; "light" and "dark". A color scheme is applied using the layout-sidebar-light or layout-sidebar-dark to the sidebar element.

Dark Menu

<div class="layout-sidebar layout-sidebar-dark">
Light Menu

<div class="layout-sidebar layout-sidebar-light">

Grid CSS

Sigma uses PrimeFlex CSS Grid throughout the samples. Although any grid library can be used, we recommend using PrimeFlex as your layout framework as it is well tested and supported by PrimeVue. PrimeFlex is available at NPM.

Customizing Styles

It is suggested to write your customizations in _overrides.scss file instead of adding them to the scss files under sass folder to avoid maintenance issues after an update.
